# AI-Engineer-Roadmap 🚀

> 30 天 AI 工程师培养计划 — 不只是会用 AI，而是理解为什么。

## 目标

我不是希望你**会用 AI**，而是希望你**理解**：
- 为什么这么设计？
- 为什么不用别的方法？
- 如果线上出了问题怎么办？

## 学习计划

| 阶段 | 天数 | 主题 |
|------|------|------|
| **基础篇** | Day 1-5 | LLM API 原理、Prompt Engineering、Token 机制、Embedding、Function Calling |
| **进阶篇** | Day 6-10 | RAG 原理与实现、向量数据库、检索策略、Reranker、知识图谱 |
| **核心篇** | Day 11-15 | Agent 架构、Tool Use、Multi-Agent、规划与推理、记忆系统 |
| **工程篇** | Day 16-20 | 模型微调、评估体系、A/B 测试、Prompt 安全、成本优化 |
| **实战篇** | Day 21-25 | SQL 增强、代码生成、多模态、工作流编排、生产部署 |
| **总结篇** | Day 26-30 | 系统设计复盘、面试冲刺、项目完善、学习方法论 |

## 每日结构

每天学习包含 8 个部分：
1. **知识体系** — 系统学习核心概念
2. **Coding** — 一步一步手写代码
3. **源码分析** — 深入设计思想
4. **面试重点** — 大厂面试题训练
5. **架构设计** — 画出系统架构
6. **思考** — 开放性问题
7. **面试记录** — 整理成面试宝典
8. **项目推进** — Risk Copilot 持续迭代

## 原则

1. 不堆知识点，建立知识体系
2. 不只告诉怎么做，还告诉为什么
3. 每个知识点都落到代码
4. 每周做一次总结复盘
